# 旧方法到新方法改进总结

本文档对比 **COD_BC 当前实现（旧方法）** 与 **EXAMPLE 中论文官方实现（新方法）**，总结鼠标采样、训练、推理三方面的差异与改进要点。EXAMPLE 即 `EXAMPLE/Counter-Strike_Behavioural_Cloning`（[Counter-Strike Deathmatch with Large-Scale Behavioural Cloning](https://arxiv.org/abs/2104.04258) 官方代码）。

---

## 1. 鼠标动作表示与预处理

| 方面 | 旧方法（COD_BC） | 新方法（Example） |
|------|------------------|-------------------|
| **存储** | 连续像素位移 `mouse_dx`, `mouse_dy`（float） | 同左，但在预处理阶段被离散化 |
| **预处理** | 仅做归一化：`dx/width`, `dy/height` → [-1, 1]，再 clip | **离散化**：将连续 (mouse_x, mouse_y) 映射到预定义档位列表中的**最近值** |
| **档位设计** | 无 | **不均匀网格**：中间密、两边疏，与人类鼠标分布一致 |
| **示例档位** | - | x: 23 档，如 `[-1000,-500,-300,...,-2,0,2,...,500,1000]`；y: 15 档，如 `[-200,-100,-50,...,0,...,200]` |

新方法在 **预处理阶段**（如 `dm_pretrain_process.py`）对每条记录的鼠标做 `mouse_preprocess(mouse_x, mouse_y)`：先 clip 到范围，再取 `mouse_x_possibles` / `mouse_y_possibles` 中**欧氏最近**的一档，得到离散标签，用于后续 onehot 与训练。

---

## 2. 模型输出与损失

| 方面 | 旧方法（COD_BC） | 新方法（Example） |
|------|------------------|-------------------|
| **鼠标输出** | 2 维连续：`tanh` → [-1, 1] | **每轴一个分类**：`n_mouse_x` 维 + `n_mouse_y` 维，各用 **softmax** |
| **鼠标损失** | **MSE**（回归） | **Categorical cross-entropy**（多分类） |
| **键位/点击** | BCE（与现有一致） | BCE（一致） |
| **论文动机** | - | 连续 + MSE 易导致“两条路选中间”的折中行为；离散 + 交叉熵可学到明确档位选择 |

新方法中模型输出为：`[keys_sigmoid, Lclick, Rclick, mouse_x_softmax, mouse_y_softmax, (value)]`，对鼠标两轴分别做多分类交叉熵。

---

## 3. 推理阶段（Test Time）

| 方面 | 旧方法（COD_BC） | 新方法（Example） |
|------|------------------|-------------------|
| **鼠标取值** | `dx = pred[0]*width`, `dy = pred[1]*height`，再 clip ±50 像素 | **argmax** 取概率最大的那一档，再查表得到**像素位移**（`mouse_x_possibles[id]` 等） |
| **移动键/鼠标** | 未区分 | **移动与鼠标**：用 argmax（确定性），避免抖动 |
| **开火/换弹/跳跃** | 点击用阈值 0.5 | 这些动作**按概率采样**（因常低于 0.5，argmax 很少触发） |
| **平滑** | 无 | **IS_SPLIT_MOUSE**：每步位移减半、同一帧内应用两次，等效提高更新频率（如 16→32 次/秒），观感更顺 |

新方法在 `onehot_to_actions` 中：`mouse_x = mouse_x_possibles[np.argmax(mouse_x_pred)]`，仅当长时间无动作（`count_inaction > 8`）时才用概率采样打破僵局。

---

## 4. 数据与训练流程（Example 侧）

- **数据格式**：.npy 存截图与元数据 → `dm_infer_actions.py` 推断动作 → `dm_pretrain_process.py` 中对鼠标做 **quantise（mouse_preprocess）** 并 **actions_to_onehot**，写入 .hdf5。
- **标签**：每帧为 `[keys_onehot, Lclicks_onehot, Rclicks_onehot, mouse_x_onehot, mouse_y_onehot]`，mouse 为 one-hot。
- **序列**：N_TIMESTEPS=96，EfficientNetB0 + ConvLSTM2D，每轴 Dense(softmax) 输出。

---

## 5. 改进要点汇总（若在 COD_BC 中采纳新方法）

1. **鼠标离散化**  
   - 定义 `mouse_x_possibles` / `mouse_y_possibles`（可先按你数据分布统计再定档位）。  
   - 采集或预处理阶段：将原始 `mouse_dx/dy` 用“最近档位”映射为离散标签。

2. **模型与损失**  
   - 鼠标改为两路 **softmax + CrossEntropyLoss**（每轴一类），去掉鼠标的 MSE 与 tanh 连续输出。

3. **推理**  
   - 鼠标：**argmax** 得到档位下标，再查表得到像素位移，用 `moveRel(dx, dy)` 等发送。  
   - 开火/换弹等：可保留或改为按概率采样。  
   - 可选：实现“位移减半、同帧应用两次”以平滑观感。

4. **尺度一致**  
   - 训练与推理统一使用**同一套档位表**与同一单位（如像素位移），避免旧方法中“归一化/反归一化 + clip”带来的尺度不一致。

按上述步骤改造后，COD_BC 的鼠标管线将与论文及 Example 实现对齐，有利于减少“折中位移”和过拟合、提升推理稳定性。
