## 整体视角：从画面到动作的完整链路

本文记录 **COD_BC 当前实现** 与 **示例 EXAMPLE（Counter-Strike_Behavioural_Cloning 官方代码）** 的完整流程，对齐从：

- **画面采样 → 预处理 → 序列构建 → 模型输入 → 训练 → 推理（动作输出 + 策略）**。

---

## 一、COD_BC：从采样到训练再到推理

### 1. 画面采样与数据写入（`src/data_collector.py`）

- **抓取源**  
  - 使用 `mss` 从主显示器抓全屏：`get_fullscreen_monitor(sct)` 得到 `monitor = {left, top, width, height}`。
  - 每帧间隔由 `config.TARGET_FPS` 控制，当前为 16 FPS。

- **主画面裁切（视野区域）**  
  - 函数：`crop_main_center_half(img, monitor)`。  
  - 含义：以屏幕中心为中心，取宽高各一半：\[H/4:3H/4, W/4:3W/4]，得到主画面原始区域。

- **小地图裁切（基于圆心定位）**  
  - 配置：`MINIMAP_CENTER_REL = (x_center, y_center, r)`，相对屏幕比例。  
  - 函数：`_minimap_center_to_roi_abs(monitor, MINIMAP_CENTER_REL)` 将圆心+半径转换为正方形框像素 `(x, y, w, h)`。  
  - 含义：
    - 圆心像素：`(x_center * W, y_center * H)`  
    - 半径像素：`r * H`  
    - 正方形边长 ≈ `2 * r * H`，居中于圆心，做边界裁剪后作为小地图裁切框。

- **图像预处理与缩放**
  - 主画面：`frame_main = preprocess_frame(img_main, MAIN_TARGET_SIZE)`  
    - `MAIN_TARGET_SIZE = (320, 90)`（宽×高），resize 后主画面为 `(90, 320, 3)`。
  - 小地图：  
    - `minimap_xywh = _minimap_center_to_roi_abs(...)` 得到 ROI；  
    - `frame_minimap = preprocess_frame_roi(img, minimap_xywh, MINIMAP_TARGET_SIZE)`；  
    - `MINIMAP_TARGET_SIZE = (90, 90)`，resize 后小地图为 `(90, 90, 3)`；  
    - `apply_circular_minimap_mask(frame_minimap)` 在裁切图中心做圆形遮罩，圆外置黑。
  - 最终单帧：`frame = np.hstack([frame_main, frame_minimap])` → **尺寸 `(90, 410, 3)`**（高×宽×通道）。

- **动作采样**
  - 鼠标：
    - 使用 `pynput.mouse` 获取屏幕坐标；  
    - 每帧根据前一帧坐标计算 `mouse_dx、mouse_dy`（像素位移，连续值）。  
  - 鼠标按键：左键、右键是否按下（0/1）。  
  - 键盘：`InputState.pressed_keys` 记录当前按下的按键集合（如 `w/a/s/d/shift/3` 等）。
  - 时间戳：每帧记录 `timestamp = now`。

- **数据写入 HDF5**
  - 每一段录制结束后（一个 `traj_i`）：
    - `states`: 堆叠所有帧 `frames` → `shape = (N, 90, 410, 3)`，`uint8`。  
    - `mouse_dx`, `mouse_dy`: `float32` 数组，长度 `N`。  
    - `mouse_left`, `mouse_right`: `int8` 数组，长度 `N`。  
    - `timestamps`: `float64` 时间戳。  
    - 可选 `keys`: 字符串矩阵，每帧一个字符串列表（为空字符串作填充）。  
    - 轨迹元数据：`monitor_left/top/width/height`、`target_fps` 等保存在 group attrs。

> 小结：采集阶段，你得到的是 **按时间顺序排列的 RGB 帧 (90,410,3)**，配上连续的鼠标位移、鼠标按键、键盘以及时间戳。

---

### 2. 序列构建与预处理（`src/data_processor.py`）

- **轨迹切片为「chunk」**（20–30 秒）
  - 函数：`build_chunked_h5(src_path, dst_path, ...)`。  
  - 目标：把长轨迹 `traj_i` 切成数个长度在 `[chunk_min, chunk_max]` 帧之间的 `chunk_j`，默认约 20–30 秒一段。  
  - 每个 chunk 拷贝同名子集：`states, mouse_dx/y, mouse_left/right, timestamps, keys`。

- **构造固定长度序列（SEQ_LEN）**
  - 数据集类：`H5SequenceDataset`。  
  - `SEQ_LEN` 来自 `config.SEQ_LEN`（当前 48），表示 LSTM 序列长度。  
  - 对每个 `chunk`，滑动窗口采样 `(start_idx → start_idx+SEQ_LEN)`，得到固定长度序列：
    - 颜色模式：`states` 维度为 4（N,H,W,3）时视为 RGB 模式。
    - 对 RGB 模式：
      - 取 `frames = states[s:e]`，归一化到 `[0,1]`：`frames/255.0`；  
      - 转为通道优先：`obs = np.transpose(frames, (0, 3, 1, 2))` → **`(T, 3, 90, 410)`**。

- **鼠标离散化（与 Example 对齐）**
  - 函数：`discretize_mouse(dx, dy)`。  
  - 档位列表来自 `config.MOUSE_X_POSSIBLES` / `MOUSE_Y_POSSIBLES`，例如：
    - X：23 档，从 -1000 到 1000 像素，中间密、两端疏。  
    - Y：15 档，从 -200 到 200 像素。
  - 对每一帧：
    - 先 clip `dx,dy` 到档位范围；  
    - 再在档位表中查找「最近的那一档」，得到下标 `(ix, iy)`；  
    - 存到 `mouse_x_class[t]`, `mouse_y_class[t]`，作为**分类标签**。

- **键盘与鼠标按键标签**
  - 鼠标左右键：  
    - `mouse_buttons[t] = [mouse_left[t], mouse_right[t]]`，float32。  
  - 键盘：
    - 行为键集合：`ACTION_KEYS = ("w", "a", "s", "d", "shift", "3")`。  
    - `encode_keyboard_multi_hot(keys)` 将每帧按键集合转为长度 6 的 0/1 多热向量。

- **最终 Dataset 输出**
  - `__getitem__` 返回一个字典：
    - `obs`: `[T, C, H, W]`，RGB 为 **(T,3,90,410)**，float32 in [0,1]。  
    - `mouse_x_class`: `[T]`，每步一个离散类别下标。  
    - `mouse_y_class`: `[T]`。  
    - `mouse_buttons`: `[T, 2]`。  
    - `keys`: `[T, 6]`。

### 3. 模型输入与结构（`src/model.py`）

- **骨干网络（EfficientNet + ConvLSTM + LSTM）**
  - 输入：`obs` 形状 `[B, T, 3, IMG_HEIGHT, IMG_WIDTH]`，当前为 `[B, T, 3, 90, 410]`。  
  - EfficientNetB0：
    - 使用 `efficientnet_b0`，截断最后的 `conv_head+bn`：`features[:-2]`，输出 192 通道特征图（中间层），减轻显存压力。  
  - 通道压缩与时间建模：
    - `feat_reduce: Conv2d(192 → convlstm_hidden=256)`。  
    - `ConvLSTM2D`：在特征图上做时序卷积 LSTM，输出 `[B, T, 256, h', w']`。  
    - `AdaptiveAvgPool2d(1)`：时序上每帧池化为 `[B, T, 256]`。  
    - 常规 LSTM：`input_size=256, hidden_size=512, num_layers=2`，输出 `[B, T, 512]`。

- **多头输出（BC 策略）**
  - 先经 `fc1: 512 → 256` + ReLU。  
  - 再分成 4 个头：
    - `fc_mouse_x: 256 → N_MOUSE_X` → `mouse_x_logits` `[B, T, N_MOUSE_X]`。  
    - `fc_mouse_y: 256 → N_MOUSE_Y` → `mouse_y_logits` `[B, T, N_MOUSE_Y]`。  
    - `fc_buttons: 256 → 2` → `mouse_buttons_logits` `[B, T, 2]`（左/右键）。  
    - `fc_keys: 256 → 6` → `key_logits` `[B, T, 6]`（WASD/Shift/3）。

- **状态（stateful 推理）**
  - `forward(obs, hidden=None)` 中 `hidden` 形如 `((conv_h, conv_c), (lstm_h, lstm_c))`。  
  - 推理时：上一步输出的 `next_hidden` 会作为下一步的 `hidden`，实现跨时间记忆。

---

### 4. 训练过程（`src/trainer.py`）

- **数据加载**
  - 使用 `create_splits(data_path, seq_len=SEQ_LEN)` 构建 train/val dataset。  
  - `DataLoader`：`batch_size=2`，`shuffle=True`，CPU worker 数为 0。

- **损失函数**
  - 鼠标：
    - `CrossEntropyLoss` 分别作用于 X/Y 两个 softmax 头：  
      - `loss_mouse_x = CE(mouse_x_logits, mouse_x_class)`  
      - `loss_mouse_y = CE(mouse_y_logits, mouse_y_class)`。
  - 鼠标按键与键盘：
    - `BCEWithLogitsLoss`：
      - `mouse_btn_loss = BCE(mouse_btn_logits, mouse_buttons)`。  
      - 如果 `USE_KEYBOARD`，再加 `keys_loss = BCE(key_logits, keys)`。
  - 总损失：
    - 若启用键盘：`loss = 0.35*(loss_mouse_x+loss_mouse_y) + 0.3*mouse_btn_loss + 0.3*keys_loss`。  
    - 否则：`loss = 0.5*(loss_mouse_x+loss_mouse_y) + 0.5*mouse_btn_loss`。

- **优化与调度**
  - 优化器：`AdamW(lr=3e-4, weight_decay=1e-5)`。  
  - 学习率调度：`StepLR(step_size=100, gamma=0.1)`。  
  - 梯度累积：`accum_steps=4`，每 4 个 batch 调用一次 `optimizer.step()`，等效 batch size=8。  
  - TensorBoard：`SummaryWriter("logs")` 记录 batch loss、epoch loss、lr。

- **早停与 checkpoint**
  - 跟踪验证集最优 `avg_val_loss`，若超过 `early_stop_patience=10` 轮无提升则早停。  
  - 保存最好模型到 `models/best_model.pt`，包含 `model_state` 和 `optimizer_state`。

> 小结：训练阶段完成了从「采样到的 `(obs, action)` 序列」到「一个条件分布 π(a|o)」的拟合——这里的策略就是 **EfficientNet+ConvLSTM+LSTM 多头网络**。

---

### 5. 推理与动作输出（`src/inferencer.py`）

- **实时画面预处理**  
  - 与采集阶段完全一致：  
    - 全屏截图 → `crop_main_center_half` → 基于 `MINIMAP_CENTER_REL` 裁切小地图 →  
    - 主画面 resize 到 `(90,320,3)`、小地图到 `(90,90,3)` → `hstack` 成 **(90,410,3)**。  
  - 再归一化到 `[0,1]`，转为 (3,90,410)，再扩维成 `(1,1,3,90,410)` 送入模型。

- **LSTM 状态维护**
  - 全局维护 `lstm_state`，初始为 `None`。  
  - 每步前向调用：`mouse_x_logits, mouse_y_logits, mouse_btn_logits, key_logits, lstm_state = model(obs, lstm_state)`。  
  - 这样模型在推理时拥有与训练一致的时序记忆（stateful）。

- **动作解码**
  - 鼠标：
    - `idx_x = argmax(mouse_x_logits)`、`idx_y = argmax(mouse_y_logits)`。  
    - 像素位移：`dx = MOUSE_X_POSSIBLES[idx_x]`，`dy = MOUSE_Y_POSSIBLES[idx_y]`。  
    - 可选平滑 `IS_SPLIT_MOUSE`：若为 True，先后两次移动 `dx/2, dy/2`，中间 sleep 一小段时间，使视角转动更顺滑。
  - 鼠标按键：
    - `mouse_btn_probs = sigmoid(mouse_btn_logits)`；  
    - 概率 > 0.5 视为按下，调用 `pyautogui.mouseDown/Up`。
  - 键盘：
    - `key_probs = sigmoid(key_logits)`；  
    - 概率 > 0.3 为「按下」，否则抬起，用 `pyautogui.keyDown/Up` 实际驱动 COD。

> 小结：推理时，策略网络接收最近一帧（加上内部记忆），输出对「下一步鼠标/按键」的分布，解码为具体像素位移与按键状态，直接作用于游戏。

## 二、Example：从采样到策略的对应流程（简要）

这里按照论文官方仓库 `EXAMPLE/Counter-Strike_Behavioural_Cloning` 的公开说明与代码结构，概括其对应链路。

### 1. 画面与元数据采样

- 环境：Counter-Strike Deathmatch。  
- 采样形式：
  - 使用专用工具/脚本将每帧的屏幕截图（固定窗口）与游戏事件/日志存入 `.npy`：  
    - 截图：RGB 图，固定窗口，直接裁在游戏画面区域。  
    - 元数据：原始鼠标、按键、时间、血量、击杀等辅助信息。

### 2. 动作推断与离线预处理

- **动作推断（`dm_infer_actions.py`）**
  - 从游戏日志 / Demo 文件中恢复「在这一帧实际发生了什么动作」，比如点击、切换武器等。  
  - 将这些动作与截图时间轴对齐，得到每帧的「ground truth 行为」。

- **预处理与量化（`dm_pretrain_process.py`）**
  - 对鼠标：
    - 读取连续的 `mouse_x, mouse_y` 像素位移。  
    - 用预先定义的 `mouse_x_possibles` / `mouse_y_possibles`（与你现在用的档位表一致）进行 **quantise**：  
      - clip 到范围后，找欧氏距离最近的档位作为类别标签。  
  - 对键盘/鼠标点击：
    - 构建 one-hot / multi-hot 表示，如 `[keys_onehot, Lclick_onehot, Rclick_onehot]`。  
  - 写出 HDF5：
    - 每帧标签为 `[keys_onehot, Lclicks_onehot, Rclicks_onehot, mouse_x_onehot, mouse_y_onehot]`。  
    - 每个样本序列长度 `N_TIMESTEPS = 96`。

### 3. 图像预处理与模型输入

- 图像：
  - 从固定窗口截图裁切游戏画面，缩放到 **(150, 280, 3)**。  
  - 训练时构造成 `[B, T, 150, 280, 3]` 或 `[B, T, 3, 150, 280]`。

- 模型：
  - Backbone：EfficientNetB0（完整或部分截断）。  
  - 时间建模：ConvLSTM2D + LSTM。  
  - 输出头：
    - 鼠标 X/Y：各一个 softmax 头，对应 `n_mouse_x` 与 `n_mouse_y`。  
    - 左右键、键盘：若干 sigmoid/BCE 输出。

### 4. 训练与策略

- 损失：
  - 鼠标：两路 cross-entropy（多分类）。  
  - 按键/点击：BCE。  
  - 可能还包含 value 头（用于后续 RL 微调），但纯 BC 时可忽略。

- 策略含义：
  - 给定最近 `T` 帧图像序列与隐藏状态，网络给出 π(a|o) 的参数：  
    - 鼠标两轴分布 + 按键/点击概率。

### 5. 推理与动作输出

- 解码：
  - 鼠标：`argmax` 选择概率最大的档位 → 查表得到像素位移。  
  - 移动与视角使用 argmax（确定性），减少抖动；  
  - 某些稀有动作（如跳跃、换弹）可能按概率采样触发。  
  - 同样支持「位移拆分两次」的平滑策略（IS_SPLIT_MOUSE）。

## 三、两套流程的对齐关系（核心差异与一致点）

- **一致点**
  - 都是 **纯行为克隆（BC）**：用专家 `(观测序列, 动作)` 拟合一个条件策略 π(a|o)。  
  - 都把 **鼠标两轴离散化** 为固定档位，并用 **softmax + CrossEntropyLoss** 训练。  
  - 都用 **EfficientNetB0 + ConvLSTM2D + LSTM** 作为视觉 + 时序 backbone。  
  - 推理时都通过 **argmax → 档位查表** 得到具体像素位移，并以一定方式进行平滑。

- **主要差异（目前 COD_BC 相对 Example）**
  - 图像尺寸：
    - Example：150×280；  
    - 你当前：主 320×90 + 小地图 90×90 → 拼接 90×410，比例更「横向拉宽」，强化水平视野与左上角小地图。
  - 序列长度：
    - Example：T=96；  
    - 你当前：T=48（减半以控制显存和训练成本）。
  - 数据来源：
    - Example：从 Demo / 官方日志推断动作，动作标签更「干净」；  
    - 你当前：直接记录自己的键鼠输入，标签近似真实人类操作，但可能包含抖动或无效动作。

整体而言，你的 **采集 → 预处理 → 序列构建 → 模型 → 训练 → 推理** 流水线，已经在设计上与 Example 对齐，只是在分辨率、序列长度和小地图布局上做了针对 COD 的定制。
