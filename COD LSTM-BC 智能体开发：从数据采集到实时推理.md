# COD LSTM-BC 智能体开发：从数据采集到实时推理

### 完整要求：COD LSTM-BC 智能体（Agent）开发

#### 核心目标

开发一个基于**LSTM-BC（行为克隆+时序建模）** 的智能体，通过模仿人类玩《使命召唤》（COD）的屏幕画面+键鼠操作，实现端到端的游戏操作模仿；核心适配 COD 连续、时序化的操作特性（如压枪、走位），保证推理延迟＜50ms（满足游戏实时性），简单场景（移动/瞄准/射击）的动作预测准确率≥80%。

## 二、核心模块开发要求

### 模块1：数据采集（Agent 训练的“专家数据集”）

#### 功能要求

实时录制人类玩 COD 时的「屏幕帧 + 键鼠操作」，生成时序化专家轨迹数据集。

#### 具体要求

1. **采集内容**：

    - 视觉状态：每帧游戏屏幕截图（仅截取游戏窗口区域，排除桌面其他内容）；

    - 动作空间：

        - 鼠标：相对移动量（x/y 轴，像素值）、是否按下左键/右键；

        - 键盘：实时按下的按键（W/A/S/D/Shift/Ctrl/空格/鼠标左键/右键，仅捕获有效游戏操作）。

2. **采集规格**：

    - 帧率：≥30fps（与游戏画面同步），保证操作时序连续性；

    - 图像预处理：实时降采样为 84×84 灰度图（减少计算量）；

    - 数据格式：单条轨迹为 `[(s_0,a_0), (s_1,a_1), ..., (s_n,a_n)]`，其中 `s` 是 84×84 灰度图（np.array），`a` 是键鼠动作字典；

    - 存储：单条轨迹≥5 分钟（避免序列过短），整体数据集≥10 小时（保证模仿效果），用 h5py 存储（格式：`/traj_{i}/states` 存储图像序列，`/traj_{i}/actions` 存储动作序列）。

3. **工具要求**：

    - 屏幕截图：用 `mss`（而非 OpenCV），仅截取游戏窗口（通过窗口句柄定位，避免截取全屏）；

    - 键鼠监听：用 `pynput` 非阻塞监听（避免卡顿游戏），记录操作的时间戳与屏幕帧一一对应。

### 模块2：数据预处理（适配 LSTM 输入）

#### 功能要求

将原始采集数据清洗、编码，转为 LSTM 可直接输入的张量格式。

#### 具体要求

1. **图像处理**：

    - 归一化：像素值缩放到 [0,1]（除以 255）；

    - 序列拼接：将连续 4 帧图像拼接为一个输入（捕捉时序信息，即 `s_t-3, s_t-2, s_t-1, s_t` 作为一组输入）；

    - 数据增强（可选）：随机翻转、亮度调整（避免过拟合）。

2. **动作编码**（核心：区分连续/离散动作）：

    - 鼠标动作（连续）：相对移动量归一化到 [-1,1]（x/y 轴分别除以屏幕宽度/高度），鼠标按键（左键/右键）编码为 0/1 离散值；

    - 键盘动作（离散）：将 W/A/S/D/Shift/空格/射击 等操作编码为 16 维独热向量（如 W=1, A=2，组合按键如 W+A=3）；

    - 动作拼接：将鼠标连续值 + 键盘独热向量拼接为完整动作向量（维度：2（鼠标xy）+2（鼠标按键）+16（键盘）=20 维）。

3. **序列处理**：

    - 轨迹切分：将长轨迹切分为固定长度的序列（如 64 帧/序列），最后不足的补 0；

    - 数据划分：按 8:1:1 划分训练集/验证集/测试集，保证轨迹完整性（同一条人类轨迹不跨集）。

### 模块3：LSTM-BC 模型搭建（Agent 核心）

#### 功能要求

构建“视觉特征提取 + 时序建模 + 动作预测”的端到端模型，适配 COD 时序操作特性。

#### 具体要求

1. **网络结构**（固定核心层，可微调参数）：

    ```Plain Text
    
    输入（4×84×84 灰度图序列）→ CNN（提取帧特征）→ LSTM（建模时序依赖）→ MLP（输出动作）
    ```

2. **各层参数**：

    - CNN 层：2 层卷积（Conv2d(1,32,8,4) → ReLU → Conv2d(32,64,4,2) → ReLU → Flatten），输出特征维度 64×9×9=5184；

    - LSTM 层：2 层 LSTM，输入维度 5184，隐藏维度 512，batch_first=True，dropout=0.2（防止过拟合）；

    - MLP 层：Linear(512, 256) → ReLU → Linear(256, 20)（输出 20 维动作向量）。

3. **输出处理**：

    - 鼠标连续值：用 tanh 激活（限制在 [-1,1]）；

    - 键盘离散值：用 softmax 激活（输出动作概率）；

    - 鼠标按键：用 sigmoid 激活（输出 0/1 概率）。

### 模块4：模型训练（Agent 学习模仿）

#### 功能要求

用监督学习训练 LSTM-BC 模型，最小化“预测动作”与“人类动作”的误差。

#### 具体要求

1. **训练配置**：

    - 损失函数：混合损失（适配连续+离散动作）：

        - 鼠标连续值：MSE 损失；

        - 键盘离散值：交叉熵损失；

        - 鼠标按键：BCE 损失；

        - 总损失：0.4×MSE + 0.5×交叉熵 + 0.1×BCE（权重可微调）；

    - 优化器：AdamW，学习率 3e-4，权重衰减 1e-5；

    - 学习率调度：训练 100 epoch 后，学习率衰减为原来的 1/10；

    - 批量大小（batch_size）：GPU 版≥32，CPU 版≥8；

    - 训练轮数：≥200 epoch，验证集损失连续 10 epoch 不下降则早停。

2. **训练监控**：

    - 用 TensorBoard 记录：训练/验证损失、各动作类型的预测准确率、学习率变化；

    - 每 10 epoch 保存一次模型 checkpoint（含权重、优化器状态），避免训练中断丢失进度。

### 模块5：实时推理（Agent 执行游戏操作）

#### 功能要求

加载训练好的模型，实时读取游戏屏幕→预测动作→模拟键鼠输入到 COD，实现自主操作。

#### 具体要求

1. **实时性**：

    - 单帧推理延迟＜50ms（含截图+模型预测+键鼠模拟）；

    - 截图频率与游戏帧率同步（30fps），避免帧丢失/重复。

2. **推理流程**：

    - 步骤1：用 mss 截取游戏窗口→预处理为 84×84 灰度图→拼接最近 4 帧；

    - 步骤2：模型预测动作向量→解码为键鼠操作（如 tanh 输出的鼠标值还原为像素移动量）；

    - 步骤3：用 pyautogui 非阻塞模拟键鼠操作（避免游戏卡顿），鼠标移动采用“平滑插值”（减少抖动）。

3. **安全机制**：

    - 加入“紧急停止”按键（如 F12），按下后立即终止推理，避免模型误操作；

    - 限制键鼠操作强度（如鼠标单次移动≤50 像素），防止操作幅度过大。

### 模块6：效果验证（Agent 性能评估）

#### 功能要求

量化评估 Agent 模仿精度，对比人类操作与 Agent 操作的一致性。

#### 具体要求

1. **评估指标**：

    - 动作预测准确率：键盘离散动作预测正确的比例（≥80%）；

    - 鼠标误差：预测鼠标移动与人类移动的平均绝对误差（MAE）≤0.1（归一化后）；

    - 推理延迟：多次测试的平均延迟（＜50ms）；

    - 人工评估：Agent 完成“移动到指定点位+瞄准固定目标”的成功率（≥70%）。

2. **测试场景**：

    - 静态场景：COD 训练场，固定视角瞄准静态靶；

    - 动态场景：COD 训练场，移动中瞄准移动靶；

    - 对比测试：同一场景下，记录人类操作与 Agent 操作的轨迹，可视化对比（用 matplotlib 绘制鼠标移动轨迹）。
